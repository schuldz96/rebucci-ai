## 1. Visual Theme & Atmosphere

Prime Coaching is a dark-mode SaaS admin panel for fitness coaches managing athletes in Brazil. The visual language is built on a near-black body (`#03101c`) that evokes the focused intensity of a training environment, punctuated by a hot pink accent (`#ff0083`) that signals energy, performance, and brand ownership. White cards float on the dark surface as elevated work areas — a classic dark-dashboard pattern that keeps data legible while immersing the coach in the application.

The type system pairs Inter (a pragmatic, workhorse grotesque used for all data and UI copy) with Onest (a slightly rounded grotesque used for headings), creating a hierarchy that feels data-dense but not cold. Weight 500 dominates interactive elements; 600-700 anchors headings and metric labels. Letter-spacing stays tight at zero to slightly negative — this is an operational tool, not a marketing page.

Surfaces use two distinct treatments: the app body and nav are pure black, while cards are either white (form dialogs, data tables) or glass — translucent white overlays on the dark background (`--card-block-bg: linear-gradient(45deg, rgba(255,255,255,0.05)...)`). Focus states and active navigation links snap to the hot pink rather than the blue, reinforcing the brand's identity anchor.

The single most distinctive visual choice is the hot pink (`#ff0083`) used as the active/accent signal. In a field of blue CTAs and dark surfaces, every pink indicator — nav active link, form popup header border, consultoria button, floating icon badge — reads as a brand signature rather than a generic interactive affordance.

**Key Characteristics:**
- Dark near-black body `#03101c` with pure black navigation `#000000`
- Hot pink `#ff0083` for brand-identity signals (nav active, popup borders, acccent badges)
- Blue `#449aff` / `#3d96ff` scale for all interactive CTAs and links
- White cards and modals float as elevated work surfaces on dark background
- Glass card blocks: white semi-transparent gradients (`rgba(255,255,255,0.05-0.2)`)
- Inter (body) + Onest (headings) type pairing — no third family in use
- Medium border-radius (4px buttons, 6px inputs, 8px cards) — utilitarian, not rounded
- Tailwind CSS v3.4 utility foundation with custom semantic var layer on top

## 3. Typography Rules

### Font Family
- **Primary UI (body, labels, data):** `Inter` — weights 100-900, self-hosted `.ttf`, `font-display: swap`. The pragmatic choice for dense data UIs; designed for screen legibility at small sizes.
- **Headings:** `Onest` — weights 100-900, self-hosted `.ttf`, `font-display: swap`. A friendly, slightly humanist grotesque that adds warmth without softening the data-dense context.
- **Monospace (code, numeric):** `ui-monospace, SFMono-Regular, Menlo, Consolas, "Courier New", monospace`. System stack — no self-hosted mono font declared.
- **Icon Font:** `fcicons` — embedded base64 font for FullCalendar calendar icons.

No OpenType feature flags are declared in the CSS. Tabular numerics (`tnum`) are not explicitly enabled — designers building financial tables should add `font-feature-settings: "tnum"` manually.

### Principles
- **Two-font system only:** Onest for headings, Inter for everything else — never swap them; never introduce a third face.
- **Weight 500 is the button neutral:** All CTA labels use weight 500. Using 400 on buttons reads as passive; using 600 reads as aggressive — 500 is the calibrated middle.
- **Small is the default:** At 14px body and 13px secondary, the system is tuned for data-dense coach dashboards, not marketing pages. Never inflate body text to 16px+ in the admin context.
- **Tight letter-spacing at all sizes:** The system does not use positive letter-spacing. Large headings track at -0.01em to -0.02em; body and UI text track at 0em. Loose tracking belongs on luxury marketing, not fitness SaaS.
- **Numeric data uses body weight, not bold:** Metric values in stats cards are weight 400-500 — bold numerics create visual noise across dashboards. Reserve 600-700 for label/heading hierarchy only.

## 7. Do's and Don'ts

**Do's**
- ✅ Use `#ff0083` (hot pink) exclusively for brand-identity signals: nav active state, popup header borders, icon accent markers. Never dilute it to a utility color.
- ✅ Use Onest weight 600-700 for headings and weight 500 for subheadings — the round grotesque character reads well on dark backgrounds at these weights.
- ✅ Use `#449aff` (--primary-600) for all interactive buttons and links — it is the established action blue.
- ✅ Keep cards white (`#ffffff`) — white cards on the `#03101c` body create the elevation without needing explicit shadows.
- ✅ Use the glass card treatment (`rgba(255,255,255,0.05-0.20)` gradients) only for dark-overlay shortcut blocks, not for form cards or data panels.
- ✅ Use Inter at 13-14px for body copy — this is tuned for information density; don't inflate to 16px in data views.

**Don'ts**
- ❌ Don't use `#ff0083` (hot pink) as the CTA background for standard buttons — CTAs use `#449aff` blue; pink is reserved for brand-identity accent moments only.
- ❌ Don't use Onest for body copy or table cells — Inter is the body font; mixing Onest into data rows disrupts the heading/data hierarchy.
- ❌ Don't use a light background (`#f9fafb` or white) for the app shell or sidebar — the entire application background is dark (`#03101c`); a light shell would invert the visual contract.
- ❌ Don't use `#033961` (dark navy) for CTA buttons — it reads as inactive/disabled at screen luminance; `#449aff` is the correct action blue.
- ❌ Don't use positive letter-spacing on any text element — the system tracks at 0em to -0.02em; positive tracking reads as old-fashioned in this context.
- ❌ Don't use weight 400 for button labels — weight 500 is the button neutral; weight 400 reads as passive and fails to signal clickability.
- ❌ Don't introduce the diagonal halftone gradient (`--gradient-bg`) in operational screens — it belongs only in lead-capture/landing surfaces.
- ❌ Don't apply `backdrop-filter: blur()` to standard cards — the glass treatment uses transparent fills, not blur. Adding blur would create visual weight inconsistent with the rest of the system.

### Breakpoints

| Name | Width | Key Changes |
|------|-------|-------------|
| `sm` | 640px | Single-column stacking for card grids |
| `md` | 768px | Sidebar collapses to icon-only or hamburger |
| `lg` | 1024px | Full sidebar + 2-3 column dashboard grid |
| `xl` | 1280px | 3-4 column grid, wider table views |
| `2xl` | 1536px | Max layout width, no further grid changes |

### Touch Targets
Minimum interactive element height: 40px (based on `10px` vertical padding on buttons with 14px font). Touch targets for nav icons should be at least 44×44px per WCAG guidelines — ensure sidebar icons have adequate hit area on tablet-width layouts.

### Collapsing Strategy
- **Navigation:** Full sidebar at lg+, collapses to hamburger or icon rail at md and below.
- **Dashboard grid:** 4 columns → 2 columns at md → 1 column at sm.
- **Tables:** Horizontal scroll container at sm; key columns pinned (student name, status).
- **Typography:** Display sizes (`40-50px`) reduce to `28-32px` at mobile; body stays `14px`.
- **Cards:** Full padding (`20px`) reduces to `12-16px` at mobile widths.

### Image Behavior
Profile pictures use `border-radius: 50%` (circular avatars) with `--card-profile-pic-border: rgba(255,255,255,0.10)` border. On small screens, avatar sizes reduce proportionally. Dashboard charts (FullCalendar integration) use responsive containers with the FullCalendar CSS custom properties (`--fc-*`).
